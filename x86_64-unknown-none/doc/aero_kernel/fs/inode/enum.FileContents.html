Enum FileContents

Source 
pub enum FileContents {
    Content(Mutex<Vec<u8>>),
    StaticContent(&'static [u8]),
    Device(Arc<DevINode>),
    Socket(Arc<dyn INodeInterface>),
    None,
}
Expand description

Enum representing the inner contents of a file. The file contents depend on the file type of the inode.

Variants§

§

Content(Mutex<Vec<u8>>)

This variant expresses a normal file (akin: A file that actually stores data in bytes) and is protected by a spin lock.

§

StaticContent(&'static [u8])

This variant is similar to the one above, except it’s read only and is backed by a static byte buffer

§

Device(Arc<DevINode>)

If the file type of the inode is FileType::Device, in that case this variant is used.

§

Socket(Arc<dyn INodeInterface>)

This variant is used to store the backing socket inode.

§

None

This file does not and cannot have any contents in bytes. This is useful in the cases of directories.
