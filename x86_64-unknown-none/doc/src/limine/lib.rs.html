4//! Rust Bindings for the limine boot protocol.
5//!
6//! # Examples
7//! An example kernel using this crate can be found
8//! [here](https://github.com/limine-bootloader/limine-rust-barebones). For
9//! smaller usage examples, see [usage](#usage).
10//!
11//! # Crate features
12//! - `uuid`: Implements `Into<uuid::Uuid>` and `From<uuid::Uuid>` for [`file::Uuid`].
13//! - `ipaddr`: Enables functions in [`file::File`] to return `Ipv4Addr`. This
14//!   is feature gated because it will only appear in stable on Rust 1.77.0, on
15//!   March 21st.
16//!
17//! # Revisions
18//! Many types in the limine boot protocol have associated revisions. These
19//! signify various added fields and changes to the protocol. For requests, if a
20//! bootloader doesn't support a revision, it will simply process it as if it is
21//! the highest revision it does support. The bootloader will only return the
22//! latest supported revision in responses. The response revision is
23//! automatically checked by the types in this crate, and types that may not be
24//! returned are wrapped in an [`Option`].
25//!
26//! In this crate, you may specify the revisions of your requests via
27//! `with_revision`, but you likely just want to use `new`. This will use the
28//! latest revision supported by this crate.
29//!
30//! # Usage
31//! The first thing you need to do is place a [`BaseRevision`] tag somewhere in
32//! your code. This tag is used to identify what revision of the protocol your
33//! kernel requires. Without this tag, the bootloader will assume revision 0,
34//! which is likely not what you want.
35//!
36//! The [`BaseRevision`] tag can be placed anywhere in your code, like so:
37//! ```rust
38//! use limine::BaseRevision;
39//!
40//! // Require version 1 or higher
41//! pub static BASE_REVISION: BaseRevision = BaseRevision::new();
42//! ```
43//!
44//! Next, you can place any requests you would like. For example, to request a
45//! larger stack (*recommended on debug Rust builds*), you can do the following:
46//! ```rust
47//! use limine::request::StackSizeRequest;
48//!
49//! // Some reasonable size
50//! pub const STACK_SIZE: u64 = 0x100000;
51//!
52//! // Request a larger stack
53//! pub static STACK_SIZE_REQUEST: StackSizeRequest = StackSizeRequest::new().with_size(STACK_SIZE);
54//! ```

67/// A tag setting the base revision supported by the kernel. Set this in your
68/// kernel in order to require a higher revision. Without this tag, the
69/// bootloader will assume revision 0.
70///
71/// The latest revision is 1.
72pub struct BaseRevision {
73    _id: [u64; 2],
74    revision: UnsafeCell<u64>,
75}
76impl BaseRevision {
77    /// Create a new base revision tag with the latest revision.
78    pub const fn new() -> Self {
79        Self::with_revision(1)
80    }
81
82    /// Create a new base revision tag with the given revision.
83    pub const fn with_revision(revision: u64) -> Self {
84        Self {
85            _id: [0xf9562b2d5c95a6c8, 0x6a7b384944536bdc],
86            revision: UnsafeCell::new(revision),
87        }
88    }
89
90    /// Check whether the given revision is supported by the bootloader.
91    pub fn is_supported(&self) -> bool {
92        (unsafe { *self.revision.get() }) == 0
93    }
94}
95unsafe impl Sync for BaseRevision {}
96unsafe impl Send for BaseRevision {}
